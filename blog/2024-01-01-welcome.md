# Welcome to SpecForge

We're excited to introduce **SpecForge** - a powerful extension system for OpenAPI v3 that adds route aliases, custom tags, and permission flags while maintaining full compatibility with the OpenAPI standard.

<!-- truncate -->

## The Problem

Modern API development requires more than what standard OpenAPI specifications provide. While OpenAPI is excellent for documenting API structure, developers often need:

1. **API Migration Support**: When evolving APIs, you need to support both old and new paths simultaneously
2. **Rich Categorization**: Simple tags aren't enough - you need metadata, colors, icons, and hierarchical organization
3. **Clear Authorization**: Security requirements should be explicit and structured, not buried in descriptions

## The Solution: SpecForge

SpecForge extends OpenAPI v3 using the standard `x-*` prefix convention, ensuring full compatibility with existing tools while adding powerful new capabilities.

### üîó Route Aliases

Define multiple paths for the same endpoint:

```json
{
  "paths": {
    "/users": {
      "get": {
        "x-route-aliases": ["/accounts", "/members"],
        "responses": { "200": { "description": "Success" } }
      }
    }
  }
}
```

This makes API migration seamless - support old and new paths simultaneously without duplicating specifications.

### üè∑Ô∏è Custom Tags

Go beyond simple tags with rich metadata:

```json
{
  "x-custom-tags": [
    {
      "name": "User Management",
      "category": "admin",
      "color": "#2563eb",
      "icon": "users",
      "description": "Operations for managing users"
    }
  ]
}
```

Perfect for organizing large APIs by domain, feature, or security level.

### üîê Permission Flags

Document authorization requirements clearly:

```json
{
  "x-permissions": {
    "required": ["users:read"],
    "optional": ["users:read:sensitive"],
    "roles": ["admin", "user-manager"],
    "scopes": ["read:users"]
  }
}
```

Make it crystal clear what permissions, roles, and scopes are needed for each endpoint.

## Getting Started

Install SpecForge:

```bash
npm install spec-forge
```

Parse your first spec:

```typescript
import { createParser } from 'spec-forge';
import apiSpec from './api-spec.json';

const parser = createParser();
parser.parseObject(apiSpec);

console.log(`API: ${parser.getInfo().title}`);
console.log(`Operations: ${parser.getAllOperations().length}`);
```

## Real-World Use Cases

### API Migration

When migrating from `/api/v1/users` to `/api/v2/users`, support both:

```json
{
  "/api/v2/users": {
    "get": {
      "x-route-aliases": ["/api/v1/users"]
    }
  }
}
```

### Permission Matrix

Generate a complete permission matrix for security auditing:

```typescript
function generatePermissionMatrix(spec) {
  const parser = createParser();
  parser.parseObject(spec);
  
  return parser.getAllOperations().map(({ path, method, operation }) => ({
    endpoint: `${method} ${path}`,
    permissions: permissionFlagsHandler.getRequiredPermissions(operation),
    roles: permissionFlagsHandler.getRequiredRoles(operation)
  }));
}
```

### Domain Organization

Organize large APIs by business domain:

```json
{
  "x-custom-tags": [
    { "name": "User Management", "category": "domain" },
    { "name": "Payment Processing", "category": "domain" },
    { "name": "Analytics", "category": "domain" }
  ]
}
```
